# 实验室招新平台前端页面规划

> 依据文档：`requirements (2).md`、`当前实现说明.md`、`当前后端流程与接口文档.md`
> 技术栈：Vue 3、Vite、TypeScript、Vue Router 4、Pinia、Element Plus、Axios、markdown-it、DOMPurify、ECharts
> 修订口径：以后端当前实现和接口文档为准；不规划当前后端未提供的自动分组、全局任务、公开资料、任务提交历史等能力。

---

## 1. 规划原则与实现边界

- 前端采用单页应用，按公开认证页、用户端、负责人工作台、管理端拆分导航与页面。
- 后端 API 统一以 `/api/v1` 为基础路径；本文后续接口表默认省略该前缀。
- 登录态由后端通过 `lab_recruit_token` HttpOnly Cookie 维护，前端不保存 JWT。
- 前端所有非安全方法请求（`POST`、`PUT`、`PATCH`、`DELETE`）必须带 `X-CSRF-TOKEN`，值来自 `XSRF-TOKEN` Cookie。
- 前端路由守卫只做体验层拦截，最终角色、时期和资源归属校验以后端为准。
- Markdown 内容统一通过安全渲染组件展示，渲染前使用 DOMPurify 做 XSS 过滤。
- 文件上传统一保存后端返回的 `attachmentFileId`，业务表单不得保存静态文件路径或临时 URL。
- 当前后端文件上传默认支持直传 `/uploads`；若部署配置 `APP_STORAGE_CHUNK_SIZE > 0`，前端需切换为顺序分片上传流程。
- 当前公告和资料列表后端未提供分页参数，前端可以做本地筛选和空态，但不能假设服务端分页。
- 任务和学习资料均为组级资源；任务没有全局任务，资料没有公开资料或按方向资料。
- 当前任务提交不保留历史版本，同一任务同一用户只有一条当前提交记录；重新提交依赖负责人或管理员打回。
- 当前没有自动分组接口，也没有拖拽移动接口；分组通过“加入分组 / 取消分组”接口逐条处理。

## 2. 角色与默认入口

| 角色 | 标识 | 默认入口 | 前端能力范围 |
| --- | --- | --- | --- |
| 新生 | `FRESHMAN` | `/app` | 报名、查看本人分组、查看公告资料、查看任务、提交任务、查看评分结果、修改密码 |
| 负责人 | `LEADER` | `/leader` | 管理自己负责的分组、未分组申请处理、组内公告、组内资料、组内任务、任务批阅、组成员查看、组数据导出 |
| 管理员 | `ADMIN` | `/admin` | 管理时期、方向、用户、报名、分组、负责人、公告、资料、任务、看板、导出、审计日志 |

登录后的跳转规则：

- `FRESHMAN`：跳转 `/app`
- `LEADER`：跳转 `/leader`
- `ADMIN`：跳转 `/admin`
- 未登录访问受保护页面：跳转 `/login?redirect=原路径`
- 已登录用户访问错误角色页面：跳回当前角色默认入口
- `/admin/login` 复用登录页组件，但登录成功后仅允许 `ADMIN` 留在管理端；其他角色回到自身默认入口

## 4. 布局规划

### 4.1 AuthLayout

用于登录、注册、找回密码、重置密码。

核心区域：

- 平台名称和当前招新时期提示
- 表单主体
- 登录、注册、找回密码入口切换
- 非报名期注册禁用提示
- 发送验证码冷却和请求过频提示

### 4.2 UserLayout

用于新生和负责人，布局可复用，菜单按角色互斥。

核心区域：

- 顶部栏：平台名称、当前时期、通知入口、用户名、角色、退出登录
- 新生菜单：首页、报名、公告、资料、我的分组、任务、成绩、个人设置
- 负责人菜单：工作台、负责分组、申请处理、公告、资料、任务、导出、个人设置
- 主内容区：页面标题、筛选区、操作区、数据区

### 4.3 AdminLayout

用于管理员。

核心区域：

- 顶部栏：平台名称、当前时期、通知入口、管理员信息、退出登录
- 侧边栏：仪表盘、时期、方向、用户、报名、分组、负责人、公告、资料、任务、导出、审计
- 主内容区：统计卡片、筛选区、表格区、详情抽屉、编辑弹窗

## 5. 路由守卫与状态初始化

### 5.1 启动加载

应用启动和路由跳转时执行：

1. 调用 `GET /meta/current-period`，写入 `metaStore.currentPeriod` 和服务端时间。
2. 对受保护页面调用 `GET /auth/me`，写入 `authStore.user`、`groups`、`leaderGroups`。
3. 需要方向树的页面调用 `GET /directions`；管理方向页调用 `GET /admin/directions`。
4. 登录后加载 `GET /notifications/summary`，用于顶部未读数量。

### 5.2 路由元信息

建议路由配置：

```ts
meta: {
  requiresAuth: true,
  roles: ['FRESHMAN'],
  layout: 'user',
  periodHint: ['REGISTRATION']
}
```

| 字段 | 作用 |
| --- | --- |
| `requiresAuth` | 是否需要登录 |
| `roles` | 允许访问的角色 |
| `layout` | 页面使用的布局 |
| `periodHint` | 前端用于按钮可用性和提示，不替代后端校验 |

### 5.3 前端时期控制

| 场景 | 前端处理 |
| --- | --- |
| 非报名期访问注册页 | 禁用注册和发送注册验证码，展示当前未开放注册 |
| 非报名期访问报名页 | 可查看历史申请，新增、编辑、撤回按钮禁用 |
| 非选拔期访问分组处理 | 分组、取消分组、拒绝申请按钮禁用 |
| 非选拔期访问任务管理 | 创建、编辑、删除、评测、打回按钮禁用 |
| 非选拔期访问任务提交 | 可查看任务，提交按钮禁用 |
| 面试期或已结束 | 保留公告、资料、成绩和历史信息查看 |

## 6. 公共状态与 API 模块

### 6.1 Pinia Store

| Store | 主要职责 |
| --- | --- |
| `authStore` | 当前用户、登录、退出、获取本人信息、角色判断、负责组和所在组 |
| `metaStore` | 当前时期、服务端时间、方向树、时期判断 |
| `notificationStore` | 未读数量、通知列表、已读操作 |
| `uiStore` | 全局加载、侧边栏折叠、面包屑、页面缓存策略 |

### 6.2 API 模块

```text
src/api
├── http.ts
├── auth.ts
├── meta.ts
├── applications.ts
├── groups.ts
├── tasks.ts
├── announcements.ts
├── materials.ts
├── uploads.ts
├── notifications.ts
├── dashboard.ts
├── exports.ts
├── admin.ts
├── leader.ts
└── audit.ts
```

`http.ts` 统一处理：

- `baseURL=/api/v1`
- `withCredentials=true`
- 从 `XSRF-TOKEN` Cookie 读取并注入 `X-CSRF-TOKEN`
- 统一响应 `ApiResponse<T>` 解包
- `40100` 清空登录态并跳转登录页
- `40300`、`40310`、`40900`、`40910`、`42200`、`42900` 的统一提示和字段错误映射
- 记录并展示 `requestId`

## 7. 公开页面规划

### 7.1 登录页

| 项目 | 内容 |
| --- | --- |
| 路由 | `/login`、`/admin/login` |
| 权限 | 匿名 |
| 主要接口 | `POST /auth/login`、`GET /auth/me`、`GET /meta/current-period` |
| 核心功能 | 邮箱密码登录、记住我、登录后按角色跳转、失败提示、登录锁定提示 |

页面状态：

- 默认态：邮箱、密码、记住我、登录按钮
- 提交中：按钮 loading，表单禁用
- 失败态：展示后端统一登录失败消息、锁定或禁用提示
- 已登录态：进入页面时根据角色自动跳转

### 7.2 注册页

| 项目 | 内容 |
| --- | --- |
| 路由 | `/register` |
| 权限 | 匿名 |
| 时期 | 仅 `REGISTRATION` 可发送注册验证码和提交注册 |
| 主要接口 | `GET /meta/current-period`、`POST /auth/send-email-code`、`POST /auth/register` |
| 核心功能 | 邮箱验证码、用户名、密码、确认密码、注册提交 |

前端校验：

- 用户名 3 到 32 位，只允许字母、数字、下划线
- 邮箱格式正确
- 密码至少 8 位，包含字母和数字
- 两次密码一致
- 验证码为 6 位数字

### 7.3 找回与重置密码页

| 项目 | 内容 |
| --- | --- |
| 路由 | `/forgot-password`、`/reset-password` |
| 权限 | 匿名 |
| 时期 | 不限制 |
| 主要接口 | `POST /auth/forgot-password`、`POST /auth/reset-password` |
| 核心功能 | 发送重置验证码、填写验证码和新密码、成功后回到登录页 |

| 权限 | `FRESHMAN` |
| 主要接口 | `GET /auth/me`、`GET /applications/summary`、`GET /announcements`、`GET /tasks`、`GET /notifications/summary` |
| 核心功能 | 当前时期、报名概览、所在分组、最新公告、近期任务、未读通知、快捷入口 |

### 8.2 报名列表与表单

| 项目 | 内容 |
| --- | --- |
| 路由 | `/app/applications` |
| 权限 | `FRESHMAN` |
| 时期 | `REGISTRATION` 可新增、编辑、撤回 |
| 主要接口 | `GET /applications`、`GET /applications/summary`、`POST /applications`、`PUT /applications/{id}`、`DELETE /applications/{id}`、`GET /directions` |
| 核心功能 | 申请列表、新增申请、编辑未分组申请、撤回申请 |

表单字段：

- 真实姓名、手机号、学院、专业、班级
- 年级、入学年份
- 一级方向、二级方向
- 自我介绍

交互规则：

- 非报名期：新增、编辑、撤回禁用，仅可查看历史申请。
- `SUBMITTED` 且未分组：报名期内可编辑和撤回。
- `GROUPED`：禁止编辑和撤回，展示进入分组结果。
- `REJECTED`、`WITHDRAWN`：只读展示。
- 同一二级方向重复提交时，提示用户修改原申请。

### 8.7 任务列表与详情

| 项目 | 内容 |
| --- | --- |
| 路由 | `/app/tasks`、`/app/tasks/:id` |
| 权限 | `FRESHMAN` 且任务所在组成员 |
| 时期 | `SELECTION` 且未截止时可提交 |
| 主要接口 | `GET /tasks`、`GET /tasks/{id}`、`GET /tasks/{id}/submission`、`POST /tasks/{id}/submission`、`GET /tasks/{id}/attachment`、`GET /tasks/{id}/submission/attachment`、`POST /uploads` |
| 核心功能 | 查看组任务、任务附件、当前提交状态、提交正文和附件、查看评分和评语 |

提交规则：

- 非选拔期禁用提交入口。
- 超过截止时间禁用提交入口。
- 正文和附件不能同时为空。
- 当前提交状态必须为 `PENDING` 才能提交。
- 已提交或已批阅后不能直接覆盖；需负责人或管理员打回后重新提交。
- 详情页只展示当前提交记录，不展示提交历史。

## 12. 文件上传规划

### 12.1 直传模式

默认按 `APP_STORAGE_CHUNK_SIZE=0B` 处理：

| 项目 | 内容 |
| --- | --- |
| 接口 | `POST /uploads?purpose=TASK_ATTACHMENT` |
| 请求类型 | `multipart/form-data` |
| 参数 | `purpose` query 参数、`file` 表单字段 |
| 返回 | `StoredFileVo`，业务表单保存 `id` |

文件用途：

- `TASK_ATTACHMENT`
- `TASK_SUBMISSION_ATTACHMENT`
- `MATERIAL_ATTACHMENT`

### 12.2 分片模式

当部署环境启用分片上传：

1. `POST /uploads/sessions` 创建上传会话。
2. `POST /uploads/sessions/{sessionId}/chunks/{chunkIndex}` 严格按顺序上传分片。
3. `POST /uploads/sessions/{sessionId}/complete` 完成上传并获得正式文件 ID。

前端限制：

- 不实现乱序分片、秒传、断点恢复。
- 上传中禁止重复提交业务表单。
- 上传失败允许重试当前步骤。

## 14. 错误与空状态规划

| 场景 | 前端表现 |
| --- | --- |
| `40100` 未登录或过期 | 清空用户状态，跳转登录页，保留 redirect |
| `40300` 无权限 | 跳转 `/403` 或在弹窗内提示无权限 |
| `40310` 当前时期不允许 | 保留页面，禁用操作，并展示时期限制提示 |
| `40400` 资源不存在 | 详情页展示不存在或已删除 |
| `40900` 数据冲突 | 表单字段或顶部错误提示，例如邮箱、用户名、分组名重复 |
| `40910` 业务状态冲突 | 提示重复报名、申请已分组、任务已提交等业务原因 |
| `42200` 数据校验失败 | 优先映射 `data.fieldErrors` 到字段级错误 |
| `42900` 请求过频 | 禁用验证码按钮，展示冷却或稍后重试提示 |
| `50000` 系统异常 | 展示通用错误、`requestId` 和重试入口 |

空状态示例：

- 暂无报名申请：报名期展示创建入口，非报名期只展示说明。
- 暂无分组：提示报名或分组尚未完成。
- 暂无公告：显示空态，不展示错误。
- 暂无资料：提示当前分组暂无学习资料。
- 暂无任务：按当前时期提示等待任务发布。
- 暂无提交：任务详情展示 `PENDING`。
- 暂无成绩：提示任务批阅后展示。
- 暂无审计日志：提示调整筛选条件。

## 17. 页面验收检查清单

- 未登录访问受保护页面会跳转登录页。
- 登录后能按角色进入正确首页。
- `/app/**` 仅 `FRESHMAN` 可访问，`/leader/**` 仅 `LEADER` 可访问，`/admin/**` 仅 `ADMIN` 可访问。
- 已登录用户访问错误角色页面时，会回到当前角色默认入口。
- 非报名期注册、发送注册验证码、报名新增、报名编辑、报名撤回不可提交。
- 非选拔期分组、拒绝申请、取消分组、任务发布、任务提交、任务批阅、任务打回不可提交。
- 负责人只能操作自己负责组的数据。
- 管理员可查看并管理全局数据。
- 报名申请状态按 `SUBMITTED / GROUPED / REJECTED / WITHDRAWN` 展示。
- 任务提交状态按 `PENDING / SUBMITTED / REVIEWED` 展示。
- 任务已提交或已批阅后不能覆盖提交，必须等待打回。
- 文件上传使用后端文件 ID，附件下载走鉴权接口。
- Markdown 内容可预览且经过安全过滤。
- 删除、撤回、拒绝、打回、撤销负责人、时期调整等关键操作有二次确认。
- 401、403、40310、409、422、429 等核心错误码有明确用户提示。
- 导出接口能正确下载 `xlsx` 文件并处理失败提示。
- 审计日志可分页查询并展示关键操作信息。
