# 实验室招新平台前端页面规划

> 依据文档：`requirements (1).md`、`API需求文档与技术栈选型.md`
> 技术栈：Vue 3、Vite、TypeScript、Vue Router 4、Pinia、Element Plus、Axios、markdown-it、ECharts
> 目标：明确前端信息架构、页面拆分、路由权限、接口依赖和开发优先级，作为前端实现基线。

---

## 1. 页面规划原则

- 前端采用单页应用，按用户端、负责人功能、管理端拆分导航与页面。
- 登录态由后端写入 `HttpOnly Cookie`，前端不保存 JWT，只维护当前用户信息。
- 前端路由守卫负责体验层面的拦截，最终权限、时期和资源归属必须以后端校验为准。
- 所有列表页默认支持分页、加载态、空态、错误态和筛选条件回填。
- 所有写操作都需要明确二次确认、按钮禁用条件和后端错误提示映射。
- Markdown 展示统一使用受控渲染组件，渲染前必须经过 XSS 过滤。
- 文件上传统一走上传组件，限制大小、扩展名、上传中状态和失败重试。

## 2. 角色与入口

| 角色 | 标识 | 默认入口 | 说明 |
| --- | --- | --- | --- |
| 新生 | `FRESHMAN` | `/app` | 报名、查看公告资料、查看任务、提交任务、查看成绩 |
| 负责人 | `LEADER` | `/leader/announcements` | 仅进入负责人工作台，处理负责组公告、任务和组员信息 |
| 管理员 | `ADMIN` | `/admin` | 管理全局配置、分组、用户、公告、资料、任务、导出 |

登录后的跳转规则：

- `FRESHMAN`：跳转到用户端首页 `/app`
- `LEADER`：跳转到负责人工作台 `/leader/announcements`
- `ADMIN`：跳转到管理端首页 `/admin`
- 未登录访问受保护页面：跳转到 `/login?redirect=原路径`
- 已登录用户访问不属于自己角色的页面：跳转到当前角色默认入口

- `/leader` 页面树使用用户端布局，仅 `LEADER` 可访问，`/leader` 默认重定向到 `/leader/announcements`。
- `/admin` 页面树仅 `ADMIN` 可访问。
- 当前前端按互斥角色处理入口和菜单：负责人不拥有普通用户端页面权限，管理员不进入负责人端。

## 4. 布局规划

### 4.1 AuthLayout

用于登录、注册、找回密码、重置密码。

核心区域：

- 平台名称与当前招新时期提示
- 表单主体
- 登录、注册、找回密码入口切换
- 注册期关闭时的明确提示

### 4.2 UserLayout

用于新生和负责人，布局复用但菜单互斥。

核心区域：

- 顶部栏：平台名称、当前时期、用户名、角色、退出登录
- 新生侧边栏：首页、报名、公告、资料、我的分组、任务、成绩、个人设置
- 负责人侧边栏：组内公告、组内任务、组员信息；任务批阅通过组内任务详情入口进入
- 主内容区：页面标题、操作区、数据区

### 4.3 AdminLayout

用于管理员。

核心区域：

- 顶部栏：平台名称、当前时期、管理员信息、退出登录
- 侧边栏：仪表盘、时期、方向、用户与报名、分组、负责人、公告、资料、任务、导出
- 主内容区：筛选区、表格区、编辑弹窗或详情抽屉

## 5. 路由守卫与状态初始化

### 5.1 启动加载

应用启动时执行：

1. 调用 `GET /api/v1/meta/current-period`，写入 `metaStore.currentPeriod`
2. 对非公开页面调用 `GET /api/v1/auth/me`，写入 `authStore.user`
3. 需要方向树的页面按需调用 `GET /api/v1/directions`

### 5.2 路由元信息

建议路由配置包含：

```ts
meta: {
  requiresAuth: true,
  roles: ['FRESHMAN'],
  layout: 'user',
  periodHint: ['REGISTRATION']
}
```

当前已实现路由按业务树绑定单一角色：`/app/**` 使用 `['FRESHMAN']`，`/leader/**` 使用 `['LEADER']`，`/admin/**` 使用 `['ADMIN']`。

字段说明：

| 字段 | 作用 |
| --- | --- |
| `requiresAuth` | 是否需要登录 |
| `roles` | 允许访问的角色 |
| `layout` | 页面使用的布局 |
| `periodHint` | 前端用于展示按钮可用性的时期提示，不代替后端校验 |

### 5.3 前端时期控制

| 场景 | 前端处理 |
| --- | --- |
| 非报名期访问注册页 | 表单禁用，提示当前未开放注册 |
| 非报名期访问报名页 | 可查看历史申请，新增、编辑、撤回按钮禁用 |
| 非选拔期访问任务提交 | 可查看任务，提交按钮禁用 |
| 非选拔期访问分组管理 | 管理按钮禁用，并保留后端错误提示 |

## 6. 公共状态与 API 模块

### 6.1 Pinia Store

| Store | 主要职责 |
| --- | --- |
| `authStore` | 当前用户、登录、退出、获取本人信息、角色判断 |
| `metaStore` | 当前时期、服务器时间、方向树、枚举字典 |
| `uiStore` | 全局加载、侧边栏折叠、面包屑、页面缓存策略 |

### 6.2 API 模块

```text
src/api
├── http.ts
├── auth.ts
├── meta.ts
├── applications.ts
├── announcements.ts
├── materials.ts
├── files.ts
├── tasks.ts
├── groups.ts
├── leader.ts
└── admin.ts
```

重点约定：

- `http.ts` 统一处理 `baseURL=/api/v1`、`withCredentials=true`、错误码提示和 401 跳转。
- 所有接口类型放入 `src/types`，分页使用统一 `PageResult<T>`。
- 上传接口独立封装进 `files.ts`，业务表单只保存返回的 `url`。

## 7. 公开页面规划

### 7.1 登录页

| 项目 | 内容 |
| --- | --- |
| 路由 | `/login`、`/admin/login` |
| 权限 | 匿名 |
| 主要接口 | `POST /auth/login`、`GET /auth/me`、`GET /meta/current-period` |
| 核心功能 | 邮箱密码登录、记住我、登录后按角色跳转、显示错误提示 |

页面状态：

- 默认态：邮箱、密码、记住我、登录按钮
- 提交中：按钮 loading，表单禁用
- 失败态：展示账号不存在、密码错误、用户禁用等后端消息
- 已登录态：进入页面时根据角色自动跳转

### 7.2 注册页

| 项目 | 内容 |
| --- | --- |
| 路由 | `/register` |
| 权限 | 匿名 |
| 时期 | 仅 `REGISTRATION` 可提交 |
| 主要接口 | `GET /meta/current-period`、`POST /auth/send-email-code`、`POST /auth/register` |
| 核心功能 | 邮箱验证码、用户名、密码、确认密码、注册提交 |

前端校验：

- 用户名 3 到 32 位，字母数字下划线
- 邮箱格式
- 密码至少 8 位，包含字母和数字
- 两次密码一致
- 验证码必填

### 7.3 找回密码页

| 项目 | 内容 |
| --- | --- |
| 路由 | `/forgot-password`、`/reset-password` |
| 权限 | 匿名 |
| 时期 | 不限制 |
| 主要接口 | `POST /auth/forgot-password`、`POST /auth/reset-password` |
| 核心功能 | 发送找回验证码、重置密码、成功后返回登录 |

## 12. 错误与空状态规划

| 场景 | 前端表现 |
| --- | --- |
| `40100` 未登录或过期 | 清空用户状态，跳转登录页 |
| `40300` 无权限 | 跳转 `/403` 或在弹窗内提示无权限 |
| `40310` 当前时期不允许 | 保留页面，禁用操作，并展示时期限制提示 |
| `40400` 资源不存在 | 详情页展示不存在或已删除 |
| `40900` 数据冲突 | 表单字段或顶部错误提示，例如邮箱重复 |
| `40910` 业务状态冲突 | 提示重复报名、申请已分组等业务原因 |
| `42200` 数据校验失败 | 优先映射到字段级错误 |
| `42900` 请求过频 | 禁用验证码发送按钮并展示倒计时 |
| `50000` 系统异常 | 展示通用错误和 requestId |

空状态示例：

- 暂无报名申请：提示创建第一份申请，非报名期则只显示说明。
- 暂无公告：显示空态，不展示错误。
- 暂无任务：按当前时期提示等待任务发布。
- 暂无分组：提示报名或分组尚未完成。
- 暂无成绩：提示任务批阅后展示。

## 16. 页面验收检查清单

- 未登录访问受保护页面会跳转登录页。
- 登录后能按角色进入正确首页。

- 新生菜单仅 `FRESHMAN` 可见，负责人菜单仅 `LEADER` 可见。
- 管理端页面仅 `ADMIN` 可访问。
- 列表页均具备分页、加载态、空态、错误态。
- 表单均具备前端基础校验和后端字段错误展示。
- Markdown 内容能正常预览，并经过安全过滤。
- 文件上传能限制类型和大小，上传失败可重试。
- 删除、撤回、撤销负责人、时期调整等关键操作有二次确认。
- 401、403、40310、409、422 等核心错误码有明确用户提示。
